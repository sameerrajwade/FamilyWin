🛠 Step 1 of 7

Install Prerequisites

Before touching Android Studio, you need 4 tools on your computer. This is a one-time setup — everything will be usable for all future apps too.

A
Node.js — the JavaScript engine
Required to run npm, install packages, and run the Expo build tools
▼
1
Download Node.js LTS
Go to nodejs.org → click the big green LTS button (not Current) → run the installer → click Next through everything
2
Verify it worked
Open Command Prompt (search "cmd" in Start menu) and type:
node --version
npm --version
You should see version numbers like v20.11.0 and 10.2.3
ℹ️
If you already have Node installed, make sure it's version 18 or higher. Run node --version to check.
B
Java Development Kit (JDK 17)
Android Studio needs Java to compile Android apps
▼
1
Download JDK 17
Go to adoptium.net → Download Temurin 17 LTS → Run the installer (Windows .msi or macOS .pkg) → Accept all defaults
2
Verify it worked
java -version
Should show: openjdk version "17.x.x"
⚠️
Use JDK 17 specifically — not JDK 21 or JDK 11. Android Gradle works best with 17.
C
Android Studio
The official IDE for Android development — includes the Android SDK and emulator
▼
1
Download Android Studio
Go to developer.android.com/studio → Download the installer → Run it → Click Next through the Setup Wizard
2
Complete the Setup Wizard
When Android Studio opens for the first time, it runs a setup wizard. Choose Standard install → It will download the Android SDK automatically (~2 GB, takes 10–20 minutes)
3
Install Android SDK 34
In Android Studio: open SDK Manager
Android Studio→Settings→Appearance & Behavior→System Settings→Android SDK
Check Android 14.0 (API 34) → Apply → OK
⚠️
Also set ANDROID_HOME environment variable to your SDK path. On Windows it's usually C:\Users\YourName\AppData\Local\Android\Sdk. Google "how to set ANDROID_HOME Windows" if unsure.
D
Firebase CLI + Expo CLI
Command-line tools for deploying functions and building the app
▼
Install both in one go:
npm install -g firebase-tools
npm install -g eas-cli
1
Log in to Firebase
firebase login
A browser window opens → sign in with your Google account → close it
✅
You're all set! Prerequisites complete. Move on to Firebase project setup.

Firebase Project Setup

Create your Firebase project, enable the services, and download the google-services.json file that connects your Android app to Firebase.

1
Create the Firebase Project
Takes about 2 minutes
▼
1
Go to Firebase Console
Open console.firebase.google.com in your browser → Sign in with your Google account
2
Create a new project
Click Add project → Name it FamilyWin → Enable Google Analytics (optional but recommended) → Click Create project
3
Add your Android app
In your new project dashboard, click the Android icon (looks like a robot)
4
Fill in the app details
Android package name: com.familywin.app
App nickname: FamilyWin
Debug signing certificate SHA-1: (get this in Step 3 below)
2
Get your SHA-1 Fingerprint
Firebase needs this to verify your app is really yours
▼
1
Unzip your project first
Unzip FamilyWin-Firebase.zip to a folder, e.g. C:\Projects\FamilyWin (Windows) or ~/Projects/FamilyWin (Mac)
2
Install packages
cd C:\Projects\FamilyWin    # or your path
npm install
This takes 2–5 minutes — downloads ~500 MB of packages
3
Generate the android/ folder
npx expo prebuild --platform android
This creates the android/ folder. This is the native Android project.
4
Get the SHA-1 fingerprint
cd android
./gradlew signingReport     # Mac/Linux
gradlew signingReport       # Windows
In the output, find the debug variant and copy the SHA1 line. It looks like: AA:BB:CC:DD:EE:FF:...
5
Paste SHA-1 into Firebase Console
Go back to Firebase Console → your Android app → Project Settings → Your apps → Add the SHA-1 fingerprint
3
Download google-services.json
The most important file — connects your app to Firebase
▼
1
Download the file
In Firebase Console → Register app → Click Download google-services.json
2
Place it in your project root AND android/app/
FamilyWin/ ├── google-services.json ← copy here (Expo reads it) ├── android/ │ └── app/ │ └── google-services.json ← also copy here (Gradle reads it) ├── lib/ ├── app/ └── package.json
🚫
Never commit google-services.json to GitHub if your repo is public. Add it to .gitignore. It contains API keys.
4
Enable Firebase Services
Authentication, Firestore, and Cloud Messaging
▼

AuthenticationAuthentication → Sign-in methodEnable Email/Password and GoogleRequired
FirestoreFirestore Database → Create databaseChoose Production mode, pick a regionRequired
Cloud MessagingCloud MessagingEnabled by default ✓Auto
AnalyticsAnalyticsEnabled when you created the projectOptional
Get Web Client ID for Google Sign-In
1
Firebase Console → ⚙️ Project Settings → General tab → scroll to Your apps → find the Web app (create one if not there) → copy the Web client ID
2
Paste into lib/googleAuth.ts
// lib/googleAuth.ts  — line 20

5
Deploy Firestore Rules + Cloud Functions
Sets up database security and the weekly reset automation
▼
1
Initialize Firebase in your project folder
cd C:\Projects\FamilyWin    # your project root
firebase init
When prompted, select (use spacebar): Firestore and Functions → press Enter → choose your FamilyWin project → accept all defaults
2
Copy the rules file
Copy firebase/firestore.rules to firestore.rules in your project root (the firebase init may have created this already)
3
Install Cloud Function dependencies
cd firebase/functions
npm install
cd ../..
4
Deploy everything
firebase deploy --only firestore:rules,functions
This takes 3–5 minutes. You'll see success messages for both the rules and the weekly reset function.
⚙️ Step 3 of 7

Configure the App

A few small edits before your first build. This is the only code you need to touch.

1
Set Your Web Client ID
Required for Google Sign-In to work
▼
1
Open lib/googleAuth.ts in any text editor
Find line 20 and replace the placeholder:
// BEFORE:
const WEB_CLIENT_ID = 'YOUR_WEB_CLIENT_ID...apps.googleusercontent.com';

// AFTER (paste your real ID):
const WEB_CLIENT_ID = '123456789-abcdef.apps.googleusercontent.com';
📍
Find your Web Client ID in: Firebase Console → ⚙️ Project Settings → General → Your apps → Web app → OAuth 2.0 client IDs
2
Verify app.json Package Name
Must match exactly what you registered in Firebase
▼
Open app.json and check the android section:
{
  "expo": {
    "android": {
      "package": "com.familywin.app",  ← must match Firebase
      "googleServicesFile": "./google-services.json"  ← path correct?
    }
  }
}
⚠️
If you change the package name here, you must also update it in Firebase Console and re-download google-services.json.
3
Create Firestore Indexes
Without these, certain queries will silently fail
▼
Go to: Firebase Console → Firestore → Indexes → Add index
Collection pathFields to indexScope
families/{id}/transactionsmemberId ASC, createdAt ASCCollection
families/{id}/transactionscreatedAt ASCCollection
families/{id}/completionsmemberId ASC, weekId ASCCollection
families/{id}/completionsmemberId ASC, wasAutoFailed ASC, completedAt ASCCollection
families/{id}/notificationConfigsenabled ASCCollection
💡
Tip: When you run the app and a query needs an index that doesn't exist, Firebase will log an error with a direct link to create it automatically. So you can also just run the app first and follow those links.
4
Regenerate the Android Folder
After any config change, always regenerate
▼
cd C:\Projects\FamilyWin
npx expo prebuild --platform android --clean
The --clean flag wipes the old android/ folder and generates a fresh one. This ensures google-services.json and all Firebase config is picked up correctly.
✅
When this completes you'll see an android/ folder in your project. That's your full Android Studio project.

The most common setup mistake
▼

apply plugin: 'com.google.gms.google-services'
If it's missing, add it. Expo prebuild should have added it automatically.

📦 Step 6 of 7

Build the Release Version

The debug build you ran on your phone has extra logging and is not suitable for Play Store. You need a signed release build. This requires a keystore — a digital certificate that proves the app is really from you.

🔑
CRITICAL: Your keystore file and its password are the keys to your app — forever. If you lose them, you can NEVER update your app on the Play Store. Back them up to multiple safe places (password manager, cloud storage, USB drive).
1
Create Your Keystore (one-time only)
This is your app's permanent digital signature
▼
1
Open a terminal / command prompt
2
Run the keytool command
keytool -genkey -v -keystore familywin-release.keystore \
  -alias familywin -keyalg RSA -keysize 2048 -validity 10000
3
Answer the prompts
Enter keystore password: Choose something strong, e.g. FamilyWin2025!
Re-enter password: same
First and last name: Your name
Organizational unit: press Enter to skip
Organization: your family name or company
City: your city
State: your state/county
Country code: US, GB, IN, etc.
Is this correct? type yes
This creates familywin-release.keystore — BACK THIS UP NOW.
2
Add Keystore to Your Android Project
Tells Gradle to use your keystore when building a release
▼
1
Copy the keystore file
Copy familywin-release.keystore into your android/app/ folder
2
Create android/keystore.properties
storeFile=familywin-release.keystore
storePassword=YourKeystorePassword
keyAlias=familywin
keyPassword=YourKeystorePassword
Replace YourKeystorePassword with the password you set above
3
Edit android/app/build.gradle
Add this to the top of the file (before the android { block):
def keystoreProperties = new Properties()
def keystorePropertiesFile = rootProject.file('keystore.properties')
if (keystorePropertiesFile.exists()) {
    keystoreProperties.load(new FileInputStream(keystorePropertiesFile))
}
Then inside the android { block, add:
signingConfigs {
    release {
        keyAlias keystoreProperties['keyAlias']
        keyPassword keystoreProperties['keyPassword']
        storeFile keystoreProperties['storeFile'] ? file(keystoreProperties['storeFile']) : null
        storePassword keystoreProperties['storePassword']
    }
}
buildTypes {
    release {
        signingConfig signingConfigs.release
        minifyEnabled true
        proguardFiles getDefaultProguardFile("proguard-android.txt"), "proguard-rules.pro"
    }
}
🚫
Add keystore.properties and *.keystore to your .gitignore file. Never commit these to version control.
3
Build the Release AAB (for Play Store)
AAB = Android App Bundle — required by Google Play
▼
1
In Android Studio menu bar:
Build → Generate Signed Bundle / APK...
2
Choose Android App Bundle
Select Android App Bundle (not APK) → Next
3
Select your keystore
Click Choose existing... → browse to your familywin-release.keystore → enter your passwords → Next
4
Choose release variant
Select release → click Finish
5
Find your AAB file
Android Studio shows a notification when done → click locate. The file is at: android/app/release/app-release.aab
📦
You now have an app-release.aab file. This is what you upload to Google Play Store.
Alternative: Build APK (for direct install / testing)
If you want an APK to install directly on phones (not via Play Store):
cd android
./gradlew assembleRelease
Output: android/app/build/outputs/apk/release/app-release.apk

5
Publishing Updates Later
How to push updates to existing users
▼
1
Bump the version code
Open android/app/build.gradle and increment versionCode:
android {
    defaultConfig {
        versionCode 2        // ← increase by 1 each release
        versionName "1.1.0"  // ← your visible version number
    }
}
2
Build new AAB → upload to Play Console → rollout
Existing users get the update automatically in 24–48 hours via Play Store auto-update.
⚠️
versionCode must always increase. You can never upload an AAB with a version code equal to or lower than the previous one.
